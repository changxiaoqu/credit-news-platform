# 产品需求文档：信用卡行业资讯平台

---

## 1. 项目概述

### 1.1 产品定位
一个简单实用的信用卡行业信息聚合网站，通过网络搜索自动获取最新行业资讯、监管政策、处罚数据、商机信息和投标信息，并按分类清晰展示，为公司业务决策提供信息支持。

### 1.2 核心价值
- **信息一站汇聚**：替代人工分散搜索，节省时间
- **支行级处罚追踪**：精确到支行的处罚数据，满足合规风控需求
- **分类清晰**：按业务主题分类，快速定位所需信息
- **智能去重**：自动识别并合并重复资讯，避免信息冗余

---

## 2. 用户角色

| 用户角色 | 核心需求 | 使用场景 |
|----------|----------|----------|
| 管理层 | 快速了解行业动态和监管风向 | 晨会汇报、战略决策 |
| 市场/销售 | 发现银行商机、招标信息 | 客户拜访前准备、投标跟进 |
| 产品/技术 | 跟踪技术应用趋势 | 产品规划、技术选型 |

| **Kimi Search（推荐）** | 中文资讯覆盖好，无需申请API Key，实时性强 | 英文资讯相对较弱 | **MVP阶段** |

#### 目标银行范围

| 银行类型 | 说明 | 示例 |

**数据过滤规则**：
- 处罚数据：只保留受罚机构为城商行或农商行的记录
- 商机/投标信息：只保留采购方为城商行或农商行的项目
- 监管政策、行业资讯：不受此限制，保持全行业覆盖

### 3.2 信息更新机制

#### 更新频率

| 信息类型 | 更新频率 | 说明 |
|----------|----------|------|
| 监管政策 | 每日 9:00一次 | 重要政策需即时推送 |
| 处罚数据 | 每日 9:00一次 | 每日更新 |
| 行业资讯 | 每日 9:00一次 | 每日更新 |
| 商机/投标 | 每日 9:00一次 | 关注截止日期临近项目 |

#### 数据去重策略
```python
# 去重规则
1. URL去重：相同source_url直接丢弃
2. 标题相似度：Levenshtein距离>0.85视为重复
3. 内容摘要相似度：TF-IDF余弦相似度>0.8视为重复
```

#### 数据存储方案

| 数据类型 | 存储方案 | 说明 |
|----------|----------|------|
| 结构化数据 | **SQLite**（MVP）/ PostgreSQL（生产） | 资讯、处罚、商机数据 |
| 全文索引 | SQLite FTS5 / Elasticsearch | 支持快速全文检索 |
| 缓存 | Redis | 热点数据、搜索结果缓存 |
| 静态文件 | 本地文件系统 | 网页模板、附件 |

## 4. 分类体系

### 4.1 一级分类（信息类型）

| 分类 | 说明 | 示例内容 |
|------|------|----------|
| 监管政策 | 监管机构发布的法规、通知、指引 | 信用卡新规、利率政策、合规要求 |
| 行业资讯 | 行业动态、市场新闻、趋势分析 | 信用卡发卡量报告、市场占有率变动 |
| 处罚数据 | 银行及其分支机构的行政处罚记录 | 央行罚单、金监总局处罚公示 |
| 商机信息 | **银行有采购需求/意向（未发布正式招标）** | 银行招募供应商意向、技术平台采购需求 |
| 投标信息 | **正式招标公告 + 中标结果** | 信用卡系统招标公告、中标公示 |

## 6. 页面功能设计

### 6.1 首页

**布局**：
```
┌─────────────────────────────────────────┐
│  Logo    监管政策 | 行业资讯 | 处罚数据   │
│          商机信息 | 投标信息    [搜索框]  │
├─────────────────────────────────────────┤
│  【今日要闻】最新5条                     │
│  【最新处罚速递】突出显示支行级别         │
│  【即将截止商机】投标截止<7天的项目       │
│       · 已截止项目置灰显示               │
│       · 按紧急程度排序（最近截止在前）  │
│  【重点政策】标记is_key_policy=true      │
└─────────────────────────────────────────┘
```

**数据更新机制**
- 显示具体最后更新时间（如：更新于: 2026-04-02 14:30）
- 支持手动刷新按钮

### 6.2 资讯列表页（各分类通用）

**筛选区**：
- 时间筛选：最近三天、最近一周、最近一月、自定义
- 搜索框：支持标题、摘要、**正文全文检索**
- **搜索结果高亮**：关键词以高亮背景显示

**列表展示**：
- 标题 + 摘要 + 发布日期 + 来源 + 关联银行
- 处罚数据特殊标识（🔴红色"罚"字标签 + 罚款金额）
- 商机信息特殊标识（🟠橙色"商机"标签 + 截止倒计时）

### 6.3 详情页
- 完整标题
- 发布信息：来源、发布日期、抓取时间
- 正文内容（保留原格式）
- **处罚数据特殊区块**（结构化展示支行级信息，含银行类型标识）
- **投标/商机信息特殊区块**（展示截止日期倒计时、银行类型标识）
- **政策信息特殊区块**（展示政策文号、效力级别、核心要点）
- **正文跳转链接**（方便用户阅读完整内容）

## 7. 非功能需求

### 7.1 性能要求

| 需求项 | 要求 |
|--------|------|
| 搜索响应 | 全文检索响应时间 < 3秒 |
| 列表加载 | 页面加载时间 < 2秒 |

### 7.2 版权合规

| 需求项 | 要求 |
|--------|------|
| 来源标注 | 所有资讯必须标注原始来源和链接 |
| 转载规范 | 摘要控制在合理范围，引导用户访问原文 |

### 7.3 容错机制

| 场景 | 处理方案 |
|------|----------|
| 搜索API失效 | 展示缓存数据，标记"数据可能不是最新" |
| 数据库故障 | 展示静态备份页面，提供核心功能 |
| 网络异常 | 本地缓存最近1000条数据，支持离线浏览 |

- 支持手机浏览器访问

- 新增半透明遮罩层，点击任意区域自动关闭侧边栏

**详情页优化**：
- 模态框高度适配手机屏幕
- 支持手势滑动关闭
